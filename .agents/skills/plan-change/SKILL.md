# Plan change

1. Read the issue, `AGENTS.md`, relevant code, tests, and current docs.
2. Explore without editing implementation files.
3. Define current behavior, acceptance criteria, and out-of-scope work.
4. Save a dated plan in `docs/ai/plans/` using the implementation-plan
   template.
5. Name exact files, risks, rollback, and verification for each step.
6. Prefer the smallest existing pattern that solves the task.
7. Leave unresolved product decisions visible for human approval.
