# Review change

1. Read `AGENTS.md`, the issue, plan, and diff.
2. Do not edit files during the review.
3. Check correctness, regressions, security, authorization, data integrity,
   architecture, accessibility, performance, and test quality.
4. For each finding provide severity, file/line, failure scenario, impact, and
   smallest safe fix.
5. Do not invent issues to look thorough.
6. End with acceptance-criteria coverage and residual risk.
