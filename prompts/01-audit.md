# Audit a repository

Read `AGENTS.md`, `IDEA.md`, architecture documentation, package scripts, and
the relevant source tree.

Do not edit files.

Produce a repository audit using
`docs/ai/templates/REPOSITORY_AUDIT.md`. Every material conclusion must cite a
file path and, where possible, a symbol or line range. Separate confirmed facts
from suspicions. Map product journeys, architecture, data/auth flow, commands,
tests, documentation conflicts, large modules, duplication, and suspected dead
code. End with a risk-ordered cleanup sequence of small tasks. Do not propose a
rewrite.
