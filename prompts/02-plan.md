# Plan a change

Read `AGENTS.md`, the issue, relevant code, tests, and current documentation.
Do not implement yet.

Create `docs/ai/plans/YYYY-MM-DD-short-name.md` from
`docs/ai/templates/IMPLEMENTATION_PLAN.md`.

The plan must include current behavior with file evidence, acceptance criteria,
out-of-scope work, steps with exact files, risk, rollback, and verification.
Prefer existing patterns and the smallest coherent solution. Call out
assumptions and unresolved product decisions instead of silently choosing.
