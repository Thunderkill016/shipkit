## Principles adopted

- keep tasks small and issue-shaped;
- separate repository modeling, diagnosis, research, planning, implementation,
  and review;
- use a compact evidence-backed project model instead of loading every file;
- report repository coverage, blind spots, confidence, and freshness;
- keep durable context, decisions, research, and outcomes in Git;
- use current primary sources for version-sensitive technical decisions;
- search for contradiction and reasons not to build;
- map opportunities before solutions;
- test important uncertain assumptions with the cheapest useful experiment;
- give agents executable pass/fail signals;
- increase human control when consequences increase;
- convert repeated mistakes and research lessons into automation.
