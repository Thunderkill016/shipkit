# Improvement memory

Store project-wide audits and improvement proposals here.

Suggested structure:

```text
YYYY-MM-DD-health-report.md
YYYY-MM-DD-short-candidate.md
```

Use the templates in `../templates/`. Update `INDEX.md` when a candidate is
rejected, parked, researched, planned, delivered, or measured.

Do not turn one audit into a giant cleanup PR. Promote one bounded candidate at
a time to a GitHub Issue.
